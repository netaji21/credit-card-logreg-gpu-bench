HOW TO PUSH THIS REPO TO GITHUB
================================
Tailored for: Windows + GitHub Desktop
Target repo:  https://github.com/netaji21/credit-card-logreg-gpu-bench

----------------------------------------------------------------------
STEP 1 - UNZIP
----------------------------------------------------------------------
Right-click credit-card-logreg-gpu-bench.zip in Downloads
-> Extract All...
-> Pick a location you will remember (Documents is fine)
You should end up with a folder named:
    credit-card-logreg-gpu-bench
containing README.md, src/, Makefile, etc.

----------------------------------------------------------------------
STEP 2 - ADD TO GITHUB DESKTOP
----------------------------------------------------------------------
Open GitHub Desktop.
Top menu:  File -> Add Local Repository...
Click  Choose...  and navigate to the extracted folder.

You will see a yellow warning that says:
   "This directory does not appear to be a Git repository."
That is expected. There is a blue link in that warning that says
   "create a repository"
Click it.

----------------------------------------------------------------------
STEP 3 - "CREATE A NEW REPOSITORY" DIALOG
----------------------------------------------------------------------
Fill in:
   Name:         credit-card-logreg-gpu-bench
   Description:  Benchmarking three GPU programming paradigms
                 (RAPIDS, OpenACC, CUDA) on credit card fraud detection.
   Local path:   (already filled in - leave alone)
   Git ignore:   None        (the repo already has its own .gitignore)
   License:      None        (the repo already has a LICENSE file)

Click  Create Repository.

----------------------------------------------------------------------
STEP 4 - FIRST COMMIT
----------------------------------------------------------------------
GitHub Desktop now shows all the files on the left as "changes".
At the bottom-left there is a commit box:

   Summary:      Initial commit: credit card fraud GPU benchmark
   Description:  (leave blank)

Click the blue  Commit to main  button.

----------------------------------------------------------------------
STEP 5 - PUBLISH TO GITHUB
----------------------------------------------------------------------
Top of the GitHub Desktop window, click  Publish repository.

A dialog appears:
   Name:                              credit-card-logreg-gpu-bench
   Description:                       (already filled in)
   Keep this code private:            *** UNCHECK THIS BOX ***
                                       (you want it public so people
                                        can actually see it)
   Organization:                      None (publish to your account)

Click  Publish Repository.

It will upload in a few seconds.

----------------------------------------------------------------------
STEP 6 - VIEW ON GITHUB
----------------------------------------------------------------------
In GitHub Desktop:
   Top menu:  Repository -> View on GitHub
   (or press Ctrl+Shift+G)

Your browser opens to:
   https://github.com/netaji21/credit-card-logreg-gpu-bench

----------------------------------------------------------------------
STEP 7 - VERIFY THE README RENDERS CORRECTLY
----------------------------------------------------------------------
On the GitHub page, scroll down and confirm:
   [x] README shows the title + badges
   [x] First chart (training time bar chart) is visible
   [x] Results table appears correctly
   [x] Second chart (productivity vs performance scatter) is visible
   [x] All folders are listed: src/, profiles/, docs/, assets/
   [x] LICENSE, CITATION.cff, Makefile at the root

----------------------------------------------------------------------
STEP 8 - ADD TOPICS (TAKES 30 SECONDS, WORTH IT)
----------------------------------------------------------------------
On the repo page, find the gear icon next to "About" on the right
sidebar.  Click it.

In the Topics field, paste:
   cuda openacc rapids gpu-computing machine-learning
   logistic-regression credit-card-fraud benchmark hpc nvidia

Click  Save changes.

Topics are how recruiters and grad-school reviewers actually find
your work via GitHub search.
